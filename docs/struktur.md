# Strukturkonzept

## Hauptnavigation

- Dashboard
- Projekte
- Webapps
- Docker-Projekte
- Netlify-Projekte
- Dokumentationen
- Dev-Tools
- Uploads
- Analysen
- Prompt-Sammlungen
- Links und Referenzen
- Aufgaben und Roadmaps
- Backups und Archive
- Systemstatus

## Datenmodell V1

V1 verwendet einfache JSON-Dateien fuer:

- Navigation
- Kategoriesammlungen
- Projektkarten
- Statusdaten

## Inhaltsprinzip

Jeder Bereich soll spaeter aus drei Ebenen bestehen:

1. Uebersicht
2. Detailansicht
3. Verknuepfte Ressourcen

## Technische Leitlinien

- keine Datenbank in V1
- keine Framework-Pflicht
- spaetere Docker- und PHP-Erweiterung muss moeglich bleiben
- statische Bereitstellung soll moeglich bleiben
