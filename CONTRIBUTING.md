# Contributing

Danke fuer das Interesse an `webapp-zentrale`.

## Grundregeln

- kleine, nachvollziehbare Aenderungen
- klare Commit-Nachrichten
- keine sensiblen Daten
- keine lokalen Backup-, ZIP- oder Cache-Dateien im Repo
- Dokumentation mitpflegen, wenn sich Struktur oder Verhalten aendert

## Arbeitsweise

1. Aenderung lokal vorbereiten
2. Struktur und Benennung sauber halten
3. README oder `docs/` aktualisieren, wenn noetig
4. Aenderung testen
5. committen und Push/Pull-Request erstellen

## Fokus in V1

Der Schwerpunkt liegt aktuell auf:

- Struktur
- Navigation
- Datenmodell
- Dokumentation

Komplexe Features sollen erst spaeter folgen.
